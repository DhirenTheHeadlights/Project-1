\doxysubsection{Detailed Description}
This class handles all the collision detection in the game. 

This class is used to represent the world in the game.

This class is used to represent the view in the game. The view is the part of the game that the player can see.

This is a child menu class for the start menu.

Simple class to handle the shop items.

This class will handle the textures for the ships in the game.

This class controls the movement and collisions of the ship class.

\doxylink{class_pirate_game_1_1_ship}{Ship} class is used to represent the ships in the game.

Derived player movement handler class.

Child class of \doxylink{class_pirate_game_1_1_ship_input_handler}{Ship\+Input\+Handler}. This class if for player ship input handling.

This class is used to represent the player ship in the game.

This class is used to represent the player in the game.

Options menu textures.

This class will handle the minimap for the game.

This class handles all the menu related functions for the game. It also contains every menu object and their respective functions.

Market texture handler.

Inherited class for land mass textures.

This class is used to represent the land masses in the game. The types of land masses include\+: islands, rocks, and shipwrecks.

This is a child menu class for the island menu.

\doxylink{class_pirate_game_1_1_interactable}{Interactable} textures for the game.

This class is the base class for all interactable objects in the game.

This class is used to represent the in-\/game HUD.

HUD textures for the game.

Singleton class to handle the wind in the game.

This class is a singleton class that contains all of the texture managers for the game.

Map for pirate game.

This class will be a singleton that will contain all of the hashmaps for the game.

Global font handler for pirate game.

Derived enemy movement handler class.

This class contains the methods for handling the input for enemy ships.

This class will handle the enemy ships in the game.

This class is almost identical to \doxylink{class_pirate_game_1_1_player_ship}{Player\+Ship}, but it is used for the enemy ships.

This class is for the interactable type of button. It is used to create the button and handle its interaction.

This class is a child class that represents an interactable drop down menu to use in the menu classes.

This singleton class contains the game state manager used for controlling which things appear on the screen

Singleton class that handles the input from the user The main functionality is to get a singlular input from the user per frame and return when needed.

This class contains all the global values used throughout the game. It is intended to be included in every file that needs access to these values.

This class is going to handle the land masses in the game. It will be used to generate the land masses and store them in a vector.

This class will contain all of the landmasses for quick access and collision detection.

The base class for all of the game menus. It is intended to be inherited from and extended to create new menus.

This class contains the methods for handling the input for the ship. This includes shooting, etc.

This class contains the inventory for the ship. It will also contain the loot/money the player has.

This is a base class for all texture handing in game. It is very important to call this class as few times as possible, as it loads all the textures into memory. Consider giving this class as a reference to other classes that need textures, rather than creating a new instance.
